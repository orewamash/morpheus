How Morpheus Works

Understanding the architecture, data flow, and when an LLM is needed.

🔬 The Core Pipeline

Every Morpheus command follows the same fundamental pipeline. The only difference is what happens at the final stage — some modes use an LLM, others use deterministic analysis.

Your Python Filescript.py → sys.settracecaptures events → Compressorgroups into chapters → Analysis EngineLLM or deterministic → Outputnarration / report / graph

Step 1 — Tracing: Python's built-in sys.settrace fires an event for every function call and return. Morpheus captures these into TraceEvent objects — function name, filename, line number, local variables, timestamps. Stdlib and site-packages are filtered out automatically.

Step 2 — Compression: Raw events are grouped into ExecutionChapter objects. Each chapter represents one function's execution — its entry, all calls it made, and its return. This reduces hundreds of events into a handful of readable chapters.

Step 3 — Analysis: Depends on the mode you choose. Some modes feed chapters to an LLM for natural language narration. Others run deterministic rules (pattern matching, statistics, ML models) directly.

🤖 Why Do Some Modes Need an LLM?

An LLM (Large Language Model) is only needed when Morpheus needs to generate human language from the execution data. The raw trace contains function names, line numbers, and variable values — useful to a computer, but meaningless to a human reading it cold.

The LLM (like Mistral running via Ollama) converts:

  • Raw trace: call calculate -> call add -> return 7
  • Narration: The calculate function was called with a=3, b=4. It called add which returned 7.

Modes like prophecy, spy, diff, and map do NOT need an LLM because they analyze the trace data with deterministic rules, pattern matching, or statistical models — no language generation required.

🌐 How Morpheus Connects to Ollama

When you install Ollama and pull Mistral, Ollama runs a local HTTP server at http://localhost:11434. Morpheus sends POST requests to /api/generate with the execution chapters as the prompt.

Morpheuschapters as prompt → POST /api/generatelocalhost:11434 → Mistralruns on your GPU/CPU → Narration textstreamed back

No data ever leaves your machine. Everything runs locally — the trace, the compression, and the LLM inference. You can optionally use OpenRouter (cloud) instead by passing -b openrouter and setting an API key.

📦 What Gets Saved to the Database?

Every run is saved to ~/.morpheus/history.db (SQLite). Stored data includes:

  • File path and mode used
  • Event count and duration in milliseconds
  • Chapters — the compressed execution structure
  • Narrations — LLM output (if narration mode was used)
  • Error — if the traced script crashed

Saved runs can be viewed with morpheus list-runs, analyzed with morpheus analyze <run-id>, or compared with morpheus diff <id1> <id2>.
